# Coverage References

This reference supports the skill's guidance on collecting and reporting code
coverage for .NET test projects.

## Official Resources

- Coverlet repository: <https://github.com/coverlet-coverage/coverlet>
- `coverlet.collector` NuGet package: <https://www.nuget.org/packages/coverlet.collector>
- ReportGenerator repository: <https://github.com/danielpalme/ReportGenerator>
- `dotnet-reportgenerator-globaltool` NuGet package: <https://www.nuget.org/packages/dotnet-reportgenerator-globaltool>
- Microsoft code coverage extension: <https://www.nuget.org/packages/Microsoft.Testing.Extensions.CodeCoverage>

## Skill-specific interpretation

- Use centralized package management for shared coverage dependencies through `test/Directory.Build.props`
- For .NET 8+, prefer the Microsoft Testing Platform coverage flow
- For .NET 6/7, use `coverlet.collector` with `XPlat Code Coverage` and `test/coverlet.runsettings`
- Keep exclusions in `test/coverlet.runsettings` or centralized test build properties instead of scattering them across test projects

## Typical report formats mentioned by the skill

- Cobertura XML for machine-readable coverage output
- HTML reports generated by ReportGenerator for human review
